Linux PrivEsc

lets us see some intro about privilege escalation,Privileges means what a user is permitted to do,Privilege escalation means escalation(gain more permissions) then what privilege is allocated. In UNIX os every single file and folder stored on the hard drive has a set of permissions associated with it, which says who is allowed to do what with the file. It is just to understand the basic fact that a user can not access (read/write/execute) files that he is not permitted to access. However, the superuser(root) can access all the files which are present on the system. In order to change any important configuration or perform any further attack, first, we need to get root access on any Linux based system. Why do we need to perform privilege escalation? - **Read/Write any sensitive file** - **Persist easily between reboots** - **Insert a permanent backdoor** Privilege escalation are two types,horizontal and vertical:- **horizontal privilege escalation** is gaining access to the user who has same permission has a current user. **Vertical privilege escalation** is gaining access to user who has higher privilege then current user.

 user@debian:~$ ls -al /etc/shadow

[Task 5] Weak File Permissions - Writable /etc/passwd

there are two files don't confuse both of the /etc/passwd file contains information about user accounts. and /etc/shadow file contain the hashes of the users accounts passwords but /etc/passwd file have more preference then /etc/shadow file . in some versions of Linux will still allow password hashes to be stored /etc/passwd . for example if you store the password hash that you generated for root user to /etc/passwd file . you can get access shell, no mater what hash in /etc/shadow file for the user root . the hashes in /etc/passwd file have higher priority 

user@debian:~$ ls -l /etc/passwd            
-rw-r--rw- 1 root root 1009 Aug 25  2019 /etc/passwd

/etc/passwd file is world-writable. so lets create user and password as our wish and add it to /etc/passwd file
